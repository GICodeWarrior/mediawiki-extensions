WikiTeX: prototype for incorporation
State: unincorporated
Prototype

  1. WikiTeX describes a modular system for incorporating rendered objects.
  2. WikiTeX introduces a new element, the rend element, which is extant in two-fold form: block and inline.
    <rend class="class" style="style" value="content" />		<!-- inline	-->

<rend class="class" style="style">				<!-- block	-->
	content
</rend>
    Quotes are optional where attributes are one-word; i.e. <rend class=chem>.
  3. The following classes have been implemented or are foreseen for implementation:
    math
    LaTeX math plus AMS-LaTeX, with commutative diagrams (source | template).
    music
    Lilypond and midi (source | template).
    chess
    LaTeX chess by Piet Tutelaers (source | template).
    chem
    PPCH-TeX chemistry macros (source | template).
    greek (foreseen)
    The conversion of beta code to polytonic Greek with pre-composed accents.
  4. Document type definition:
    <!ELEMENT REND - 0 (%block;|%inline;)* +(#PCDATA)>
<!ATTLIST REND
	class   NAME    #REQUIRED
	value   CDATA   #IMPLIED
	style	CDATA	#IMPLIED
	>

  6. Horizon:
    • Transition from a modular (<rend class=math>) to a token tag system (<math>).  (Awaiting consensus on tokens, whether XML <math>; wiki-esque [[[math]]]; or allomorph [!!...!!].)
      • Though tokens offer some convenience, there are certain expansibility concerns: in order to add new modules, arbitrary and unintuitive tags have to be fabricated (allomorph); or there may be semantic problems (XML; <math>, for instance, being the root element in MathML).  (Consensus dictates XML-style tags.)
