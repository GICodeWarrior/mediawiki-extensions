Problems when running scripts

If you (1) are using Windows and (2) have perl-scripts and (3) you are used to start Xerver with "StartXerver.exe" you might notice that everytime someone runs a perl-script with your webserver a dos-prompt will popup and then close down (very quickly).
If you have this problem, the solution is to:

  1. Create an empty text file in your Xerver directory (call it "StartXerverShortcut.txt").
  2. Open the text file, write "java Start" and save the file.
  3. Now rename the file to "StartXerverShortcut.bat".
  4. Done! Run "StartXerverShortcut.bat" every time you want to start Xerver. (When you run "StartXerverShortcut.bat" a dos-prompt will open, don't close it until you want to shut down Xerver).
